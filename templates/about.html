Algorithm Overview

Preemptive Shortest Job First (SJF), also known as Shortest Remaining Time First (SRTF), is a scheduling strategy where the CPU is allocated to the process that is closest to completion.

"If a new process arrives with a CPU burst length less than the remaining time of the current executing process, the current process is preempted."

Execution Flow

// Pseudo-Logic for SRTF Scheduler
while (processes_remaining) {
    current = find_shortest_remaining_time(ready_queue);
    execute(current, 1_time_unit);
    if (new_arrival.burst < current.remaining_time) {
        preempt(current);
        current = new_arrival;
    }
}

Advantage

Produces the minimum average waiting time for a given set of processes. It ensures high-turnover for small tasks.

Limitation

Requires knowledge of future CPU burst times, which is difficult to estimate. Long processes may face starvation.
