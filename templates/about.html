System Documentation

Preemptive Shortest Job First (SJF) scheduling, often called Shortest Remaining Time First (SRTF), is a dynamic CPU management strategy where the scheduler selects the process with the smallest remaining execution time to run next. Unlike the non-preemptive version, this algorithm can interrupt a currently running process if a new task arrives with an even shorter burst time. In the context of dynamic partitioning, this ensures that short processes do not get stuck behind long-running tasks, minimizing the Average Waiting Time. However, it requires the system to constantly monitor and compare remaining times, which increases overhead, and it carries the risk of starvation for longer processes if a steady stream of short tasks keeps entering the queue.

Advantage

Optimum average waiting time for a given set of processes.

Risk

Starvation of longer processes if short tasks arrive continuously.
